# Magic Numbers Documentation

This document serves as a reference for magic numbers found throughout the codebase that have not been refactored into explicit constants. They generally fall into categories like vector math thresholds, timing defaults, input hashes, or debug sizing.

## Player / Movement
* `PlayerController.cs`: Vector squared magnitude comparisons (`0.01f`, `0.0001f`) are used as minimum thresholds to check if the player has valid movement input to avoid drifting or locking facing direction.
* `PlayerMotor.cs`: `0.70710678f` is used. This represents $1/\sqrt{2}$, likely used for normalizing diagonal movement vectors.
* `DashAbility.cs`: Direction magnitude thresholds (`0.01f`) to prevent dashing with zero input.

## Audio
* `MusicManager.cs`: `0.0001f` is used extensively in crossfading logic to prevent division by zero when calculating `fadeDuration` and normalized time `tGlobal`.

## World Generation
* `WorldGenRunner.cs`: Hash inputs like `12345`, `67890`, and `9999` are deterministic seeds for structural noise layers. Threshold comparisons (e.g. `< 0.00001f`) exist to prevent division by zero in geometry math.
* `TilePathfinder.cs`: Costs (`1.4142f`) represent the mathematical approximation of $\sqrt{2}$ for diagonal traversal.

## Enemy AI & State Machines
* Various `Wolf` and `Badger` behaviors (`WolfChaseSO.cs`, `BadgerWalkSO.cs`): Magnitude checks (`0.0001f`) exist to ensure direction vectors are valid before normalization. Delay counters and loop boundaries (e.g., trying to find paths up to `10` times) define fallback limits.
* `EnemyDoubleTestSO.cs`: Bullet lifetime set to `5f` seconds and side spread offset to `0.05f` for tuning attack spread.

## Player Animation & Weapons
* `PlayerAnimationController.cs`: `0.05f` crossfade durations are hardcoded as a fallback blend time between animation states.
* `BowSO.cs` / `PlayerBowController.cs`: Projectile vector length thresholds (`0.0001f`) are present to avoid dividing by zero or sending arrows with no direction.

## Input System
* `InputSystem_Actions.cs`: Contains autogenerated UUID strings matching input map elements. These represent the Unity Input System's internal data structure.
